## Story

**As a** usuário premium do Launch Studio PRO,
**I want** criar, duplicar, arquivar e apagar projetos de lançamento,
**so that** eu possa gerenciar meu workspace com controle total do ciclo de vida dos projetos (PRD §2 ações, §11.2).

## Acceptance Criteria

1. API route `POST /api/launch-studio-pro/projects` cria um novo projeto com campos: `title`, `target_audience`, `initial_briefing`, `user_id` do usuário autenticado; status inicial `draft`.
2. API route `GET /api/launch-studio-pro/projects/[id]` retorna projeto por ID, validando que `user_id` pertence ao usuário atual (RLS).
3. API route `PATCH /api/launch-studio-pro/projects/[id]` atualiza campos editáveis (`title`, `target_audience`, `initial_briefing`, `launch_type`, `status`) do projeto.
4. Ação "Arquivar": atualiza `status='archived'` via PATCH (não apaga dados).
5. Ação "Soft Delete": seta `deleted_at = NOW()` via PATCH ou DELETE route — projeto não aparece mais na lista (PRD §11.2).
6. Ação "Hard Delete": agendada automaticamente 30 dias após `deleted_at` (pode ser implementada como cron job ou nota de dívida técnica documentada — PRD §11.2).
7. Ação "Duplicar": cria novo projeto com mesmos dados mas novo `id`, `slug`, `created_at`, `status='draft'`, sem copiar dossiês.
8. Todas as operações validam `user_id` server-side (nunca confiar no client).
9. Respostas de erro apropriadas: 401 (não autenticado), 403 (não premium ou projeto de outro usuário), 404 (projeto não encontrado), 422 (dados inválidos).
10. `npm run lint` e `npm run typecheck` passam.
