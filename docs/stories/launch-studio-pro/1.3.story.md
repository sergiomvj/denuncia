## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** habilitar Row Level Security (RLS) nas tabelas `launch_projects`, `launch_interview_answers` e `launch_dossiers`, com políticas de isolamento por usuário e a função `require_premium()`,
**so that** nenhum usuário acesse dados de outro usuário e o gate premium seja reforçado também na camada do banco (PRD §8.3).

## Acceptance Criteria

1. RLS habilitado nas tabelas `launch_projects`, `launch_interview_answers` e `launch_dossiers` com `ALTER TABLE ... ENABLE ROW LEVEL SECURITY`.
2. Política `launch_projects_user_isolation` criada: `USING (user_id = auth.uid())` para todas as operações (PRD §8.3).
3. Política `interview_answers_user_isolation` criada: `USING (project_id IN (SELECT id FROM launch_projects WHERE user_id = auth.uid()))` (PRD §8.3).
4. Política `launch_dossiers_user_isolation` criada: `USING (project_id IN (SELECT id FROM launch_projects WHERE user_id = auth.uid()))` (PRD §8.3).
5. Função SQL `require_premium()` criada: retorna `boolean`, verifica `EXISTS (SELECT 1 FROM users WHERE id = auth.uid() AND is_premium = TRUE)` (PRD §8.3).
6. Teste de isolamento: usuário A não consegue ler projetos do usuário B via query direta ao Supabase.
7. Teste de isolamento: usuário não-premium não consegue inserir em `launch_projects` quando RLS está ativo.
8. `npx prisma validate` e `npm run typecheck` passam após as alterações.

## Dev Notes

### Contexto brownfield
Supabase/PostgreSQL com `auth.uid()` disponível nas políticas RLS. Padrões de RLS existem em `prisma/schema.prisma` para outros modelos — verificar antes de criar.

### SQL das políticas (PRD §8.3)

```sql
-- launch_projects
ALTER TABLE launch_projects ENABLE ROW LEVEL SECURITY;
CREATE POLICY launch_projects_user_isolation
  ON launch_projects FOR ALL
  USING (user_id = auth.uid());

-- launch_interview_answers
ALTER TABLE launch_interview_answers ENABLE ROW LEVEL SECURITY;
CREATE POLICY interview_answers_user_isolation
  ON launch_interview_answers FOR ALL
  USING (
    project_id IN (
      SELECT id FROM launch_projects WHERE user_id = auth.uid()
    )
  );

-- launch_dossiers
ALTER TABLE launch_dossiers ENABLE ROW LEVEL SECURITY;
CREATE POLICY launch_dossiers_user_isolation
  ON launch_dossiers FOR ALL
  USING (
    project_id IN (
      SELECT id FROM launch_projects WHERE user_id = auth.uid()
    )
  );

-- Função require_premium
CREATE OR REPLACE FUNCTION require_premium()
RETURNS boolean AS $$
  SELECT EXISTS (
    SELECT 1 FROM users
    WHERE id = auth.uid() AND is_premium = TRUE
  );
$$ LANGUAGE sql STABLE;
```

### Dependência
Depende de 1.1 (tabelas devem existir antes do RLS). Não depende de 1.2 mas é complementar ao gate.
