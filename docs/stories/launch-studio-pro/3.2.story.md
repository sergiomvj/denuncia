## Story

**As a** usuário premium do Launch Studio PRO,
**I want** as 10 etapas da entrevista PLF configuradas com todos os seus campos, dropdowns e multi-selects,
**so that** eu possa responder perguntas específicas sobre avatar, oferta, tipo de lançamento, sequência PLF, escassez e estímulos mentais (PRD §4.2).

## Acceptance Criteria

1. Etapa 1 (Aprofundamento do Avatar): campos para dores, sonhos, medos, objeções antecipadas, linguagem e canais do avatar (PRD §4.2 Etapa 1).
2. Etapa 2 (A Transformação): campos para estado atual→desejado, prazo, prova de transformação, consequência de não comprar (PRD §4.2 Etapa 2).
3. Etapa 3 (A Oferta Irresistível): campos para produto principal, até 5 bônus, garantia, preço âncora e preço de lançamento, formas de pagamento (PRD §4.2 Etapa 3).
4. Etapa 4 (Tipo de Lançamento): dropdown ou radio com as 5 opções: Seed Launch™, Lançamento Interno, Lançamento Conjunto, Lançamento Perene, Business Launch Formula™ (PRD §4.2 Etapa 4).
5. Etapa 5 (Pré-Pré-Lançamento): campos para tamanho da lista, canais disponíveis (checkboxes: e-mail, IG, YouTube, etc.), top 3 perguntas, tom de voz (PRD §4.2 Etapa 5).
6. Etapas 6, 7 e 8 (PLC 1, 2, 3): campos específicos de cada PLC conforme PRD §4.2 Etapas 6-8.
7. Etapa 9 (Escassez & Fechamento): tipo de escassez (dropdown: preço sobe / bônus somem / oferta encerra), duração da abertura, e-mails de fechamento (PRD §4.2 Etapa 9).
8. Etapa 10 (Estímulos Mentais & Parcerias): multi-select de estímulos (Autoridade, Reciprocidade, Confiança, Expectativa, Empatia, Eventos, Comunidade, Escassez, Comprovação Social) + sim/não para Lançamento Conjunto + nicho de parceiros (PRD §4.2 Etapa 10).
9. Cada etapa renderiza corretamente no wizard (conectado ao engine de 3.1).
10. Todos os campos usam componentes do Design System v2 (inputs, textareas, dropdowns, checkboxes).
11. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Estrutura de configuração das etapas

```ts
// config/interview-steps.ts
export const INTERVIEW_STEPS: WizardStep[] = [
  { stepNumber: 1, stepSlug: 'avatar', title: 'Aprofundamento do Avatar', component: AvatarStep },
  { stepNumber: 2, stepSlug: 'transformation', title: 'A Transformação', component: TransformationStep },
  // ... até 10
];
```

### Estímulos mentais (etapa 10, PRD §4.2)
Lista exata conforme PRD: Autoridade · Reciprocidade · Confiança · Expectativa · Empatia · Eventos · Comunidade · Escassez · Comprovação Social.

Slugs para o banco: `authority | reciprocity | trust | anticipation | empathy | events | community | scarcity | social_proof` (mesmos de `launch_triggers_map.trigger_name`).

### Tipos de lançamento (etapa 4, PRD §4.2)
- Seed Launch™
- Lançamento Interno (lista própria)
- Lançamento Conjunto (parceiros)
- Lançamento Perene (evergreen)
- Business Launch Formula™ (múltiplas ofertas encadeadas)

### Dependência
Depende de 3.1 (wizard engine deve existir para conectar as etapas).

### Article IV — No Invention
Cada campo desta story rastreia para PRD §4.2 (etapas 1-10). Não adicionar campos que não estejam no PRD.

### Testing

- Renderizar cada uma das 10 etapas e verificar que todos os campos aparecem
- Preencher etapa 10 e verificar que dados são serializados corretamente para jsonb
- Verificar que multi-select de estímulos salva array de slugs
