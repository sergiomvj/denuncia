## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** implementar PII detection básica e sanitização de logs para garantir que API keys e dados sensíveis nunca apareçam em logs, prompts ou artefatos exportáveis,
**so that** o app esteja em conformidade com as diretrizes de segurança do PRD (PRD §11.5, params §12).

## Acceptance Criteria

1. Função `sanitizeLog(text: string): string` implementada: remove/mascara padrões de API key (ex: `sk-...`, `Bearer ...`, strings de 40+ chars alfanuméricos em posições de credencial).
2. Todos os logs de `premium_llm_calls` passam por `sanitizeLog` antes de serem persistidos em `input_summary` ou `output_summary` (quando aplicável em `premium_tool_calls`).
3. Nenhuma API key (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, etc.) aparece em logs de erro ou output de console do servidor.
4. Função `detectPII(text: string): boolean` implementada com detecção básica: e-mails, CPF (formato 000.000.000-00), número de cartão de crédito (padrão Luhn).
5. Se PII detectado nos outputs dos agentes LLM antes de salvar em banco, gerar warning de log (não bloquear, apenas alertar — MVP).
6. Variáveis de ambiente sensíveis (`ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_API_KEY`, `PERPLEXITY_API_KEY`) carregadas apenas no server via `process.env`, nunca expostas ao cliente.
7. Testes unitários de `sanitizeLog` e `detectPII` passam.
8. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Padrões de sanitização

```ts
const API_KEY_PATTERNS = [
  /sk-[a-zA-Z0-9]{20,}/g,           // OpenAI
  /anthropic-version/gi,              // Anthropic header
  /Bearer\s+[a-zA-Z0-9\-_.]{20,}/g, // Generic Bearer
  /[a-zA-Z0-9]{40,}/g,               // Long alphanumeric (genérico)
];

export function sanitizeLog(text: string): string {
  let result = text;
  API_KEY_PATTERNS.forEach(pattern => {
    result = result.replace(pattern, '[REDACTED]');
  });
  return result;
}
```

### PII básico (MVP)

```ts
const PII_PATTERNS = [
  /\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b/,  // email
  /\d{3}\.\d{3}\.\d{3}-\d{2}/,  // CPF
  /\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}/,  // Cartão
];
```

### Dependência
Depende de 4.2 (roteador com logging). Deve ser integrada antes de qualquer chamada LLM em produção.

### Nota: PRD §16
"Não enviar dados sensíveis para múltiplos provedores sem necessidade." Verificar que apenas os dados estritamente necessários são enviados nos prompts.
