## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** implementar o `fidelity_reviewer` que calcula o `fidelity_score` do dossiê e gravar o dossiê completo em `launch_dossiers`/`launch_phases` com versionamento,
**so that** o usuário receba um dossiê fiel à PLF com número de versão incrementado e um momento de celebração na entrega (PRD §6, §8.2, §12 UX).

## Acceptance Criteria

1. `fidelity_reviewer` implementado: recebe dossiê completo (output de 5.3, 5.4, 5.5, 5.6), usa `ModelRouter` com `phase_role='reviewer'`, produz `fidelity_checklist` (itens binários verificáveis) e `fidelity_score` (0-100).
2. `fidelity_checklist` materializa os itens do schema PRD §9.2 como verificações binárias (ex: "Pré-Pré presente: sim/não", "PLC 1 tem theme=Oportunidade: sim/não").
3. `fidelity_score` calculado como porcentagem de itens do checklist marcados como verdadeiros.
4. Dossiê completo e validado gravado em `launch_dossiers` com versão incrementada (`version = previous + 1` ou `1` se primeiro).
5. Fases do dossiê gravadas em `launch_phases` (um registro por fase PLF: pre_pre, plc1, plc2, plc3, cart_open, cart_close) com `scheduled_date` calculado a partir do cronograma gerado.
6. `launch_projects.fidelity_score` atualizado com o score calculado.
7. `launch_projects.status` atualizado para `'ready'` após gravação bem-sucedida.
8. UI exibe momento de celebração após entrega (ex: animação de confetti, mensagem "Seu Dossiê de Lançamento está pronto! 🎉") — PRD §12 UX.
9. Testes de integração para o reviewer e persistência.
10. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Fidelity score — itens binários (R5)
[AUTO-DECISION] Checklist de fidelidade materializado como itens binários verificáveis por código, não pelo LLM. O LLM (`fidelity_reviewer`) pode sugerir ajustes textuais, mas o score é calculado programaticamente sobre o JSON do dossiê.

### Versionamento

```ts
const lastVersion = await prisma.launchDossier.findFirst({
  where: { projectId },
  orderBy: { version: 'desc' },
  select: { version: true }
});
const nextVersion = (lastVersion?.version ?? 0) + 1;
```
