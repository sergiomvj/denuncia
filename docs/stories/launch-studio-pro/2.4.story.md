## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** aplicar o limite de projetos por plano (Starter: 3, Pro: 25, Agency: ilimitado) ao criar novos projetos,
**so that** usuários que atingiram o limite vejam um modal de upgrade sem que seus projetos existentes sejam apagados (PRD §10, §12).

## Acceptance Criteria

1. Ao tentar criar um novo projeto via `POST /api/launch-studio-pro/projects`, o sistema verifica o `premium_plan_slug` do usuário e o limite correspondente em `user_premium_limits`.
2. Se o usuário atingiu o limite do plano (contagem de projetos ativos não-deletados >= max_projects_per_month): a API retorna 429 com mensagem clara.
3. Frontend exibe modal de upgrade ao receber erro 429, com CTA para upgrade de plano.
4. Projetos existentes NÃO são apagados quando o limite é atingido (PRD §10).
5. Usuários com `plan_slug = null` (premium sem plano definido) são tratados como Starter.
6. Usuários Agency (`max_projects_per_month = -1`) não são bloqueados.
7. Contagem considera apenas projetos com `deleted_at IS NULL` (projetos soft-deleted não contam para o limite).
8. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Lógica de verificação de limite

```ts
async function checkProjectLimit(userId: string) {
  const user = await prisma.user.findUnique({ where: { id: userId } });
  const planSlug = user.premiumPlanSlug ?? 'starter';
  const limits = await prisma.userPremiumLimits.findFirst({ where: { planSlug } });

  if (limits.maxProjectsPerMonth === -1) return; // Agency: sem limite

  const activeProjects = await prisma.launchProject.count({
    where: { userId, deletedAt: null }
  });

  if (activeProjects >= limits.maxProjectsPerMonth) {
    throw new LimitExceededError('project_limit', limits.maxProjectsPerMonth);
  }
}
```
