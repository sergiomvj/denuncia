## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** implementar o `sequence_planner` que gera o cronograma completo PLF com roteiros de PLC 1, 2 e 3, pré-pré, abertura e fechamento,
**so that** o dossiê contenha a sequência completa fiel à PLF (PRD §6, §9.2 sequence).

## Acceptance Criteria

1. `sequence_planner` implementado em `lib/launch-studio-pro/agents/sequence-planner.ts`: recebe briefing completo (projeto + todas as respostas da entrevista), usa `ModelRouter` com `phase_role='planner'`.
2. Saída cobre todas as 6 fases: `pre_pre` (strategy, email_script, survey_questions), `plc1` (theme=Oportunidade, script, key_points, cta), `plc2` (theme=Transformação, script, teaching, case_study), `plc3` (theme=Propriedade, script, offer_tease, scarcity_seed), `cart_open` (email_script, sales_page_outline), `cart_close` (emails[], scarcity_amplification) — conforme PRD §9.2.
3. Saída validada contra schema `sequence` de 5.2.
4. Themes de PLC 1/2/3 são fixos conforme PLF: Oportunidade, Transformação, Propriedade (nunca alterados pelo LLM).
5. API route `POST /api/launch-studio-pro/projects/[id]/generate/sequence` dispara o agente e persiste `payload.sequence` em `launch_dossiers`.
6. Testes de integração com mock do ModelRouter.
7. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Schema de sequência (PRD §9.2)

```ts
sequence: {
  pre_pre: { strategy: string, email_script: string, survey_questions: string[] },
  plc1: { theme: "Oportunidade", script: string, key_points: string[], cta: string },
  plc2: { theme: "Transformação", script: string, teaching: string, case_study: string },
  plc3: { theme: "Propriedade", script: string, offer_tease: string, scarcity_seed: string },
  cart_open: { email_script: string, sales_page_outline: unknown[] },
  cart_close: { emails: unknown[], scarcity_amplification: string }
}
```

### Themes PLF — fixos
O LLM pode esquecer/alterar o theme. Após receber a saída, fixar os values:
```ts
output.sequence.plc1.theme = 'Oportunidade'; // override
output.sequence.plc2.theme = 'Transformação';
output.sequence.plc3.theme = 'Propriedade';
```

### Dependência
Depende de 5.2 (schema e validação). Os agentes 5.3 (avatar/offer) são paralelos — sequence_planner pode rodar após ambos ou com contexto de briefing direto.

### Testing

- Mock retornando sequência válida: `runSequencePlanner` sem erro
- Mock sem `plc2`: ValidationError
- Verificar que themes são fixados corretamente após output do LLM
