## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** implementar o roteador de modelos que seleciona o provedor/modelo correto por fase PLF com perfil `quality_first`,
**so that** cada agente operacional use o LLM mais adequado à sua função sem depender de nome hardcoded (PRD §5; params §7.4).

## Acceptance Criteria

1. Módulo `ModelRouter` implementado em `lib/launch-studio-pro/model-router.ts` (ou equivalente).
2. Roteamento seleciona o modelo por `phase_role` consultando `premium_model_configs` (não hardcoded no código — PRD §5, params §7.4).
3. Perfil de roteamento `quality_first` aplicado: prioriza capacidade sobre custo.
4. Suporte a 4 provedores: Anthropic (SDK `@anthropic-ai/sdk`), OpenAI (`openai`), Google (`@google/generative-ai`), Perplexity (via API HTTP ou SDK).
5. Cada chamada de modelo registra início do `premium_app_run` se ainda não iniciado.
6. Interface unificada de chamada: `router.call({ phaseRole, systemPrompt, userPrompt, projectId, runId })` retorna `{ content: string, usage: TokenUsage }`.
7. Configuração carregada do banco (não de variáveis de ambiente direto) com cache em memória por request.
8. `npm run lint`, `npm run typecheck` e testes unitários do roteador passam.

## Dev Notes

### Atenção de escopo (STORIES-BREAKDOWN R1)
A lib atual `lib/sextou-tools-pro/generation.ts` é single-shot OpenAI. Esta story é um CREATE justificado (capacidade nova multi-provider). NÃO adaptar a lib existente — criar módulo novo em `lib/launch-studio-pro/`.

### Interface do roteador

```ts
// lib/launch-studio-pro/model-router.ts
export interface CallParams {
  phaseRole: PhaseRole;
  systemPrompt: string;
  userPrompt: string;
  projectId: string;
  runId: string;
}

export interface CallResult {
  content: string;
  usage: { inputTokens: number; outputTokens: number };
  provider: string;
  model: string;
}

export type PhaseRole =
  | 'interview_coach' | 'planner' | 'copywriter'
  | 'avatar_researcher' | 'objection_handler' | 'reviewer' | 'exporter';
```

### API Keys — servidor apenas
```ts
// lib/launch-studio-pro/providers/anthropic.ts
const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY }); // servidor only
```
Nunca exportar keys, nunca logar em produção.

### Dependência
Depende de 4.1 (tabelas e seed de configs existem).

### Spike recomendado (R1)
STORIES-BREAKDOWN recomenda spike de validação dos contratos de provider antes da Onda 3. Registrar qualquer incompatibilidade de API encontrada como nota no Dev Agent Record.

### Testing
