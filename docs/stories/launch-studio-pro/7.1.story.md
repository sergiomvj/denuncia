## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** criar a tabela `premium_cost_ledger` e implementar estimativa de custo antes de cada geração e registro do custo real após,
**so that** o sistema exiba o custo estimado ao usuário antes de gerar e mantenha ledger auditável de todos os custos (PRD §10, params §11.9).

## Acceptance Criteria

1. Tabela `premium_cost_ledger` criada com campos: `id`, `user_id`, `app_slug`, `project_id`, `run_id` (FK para `premium_app_runs`), `estimated_cost_usd`, `actual_cost_usd`, `created_at`.
2. Estimativa de custo (~$0.68 para dossiê completo, baseada no PRD §10) calculada antes da geração via `estimateCost()` de 4.4 e exibida no modal de confirmação de 3.5.
3. Após conclusão da run, `actual_cost_usd` é atualizado com o custo real de `premium_app_runs.total_cost_usd`.
4. Registro inserido em `premium_cost_ledger` para toda run (sucesso ou falha), com `estimated_cost_usd` e `actual_cost_usd` quando disponível.
5. Alerta admin quando custo diário total > $50 ou custo de um usuário no mês > $20 (PRD §10).
6. Alerta implementado como log de servidor com nível WARN (email/webhook pode ser enhancement pós-MVP).
7. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Tabela `premium_cost_ledger`

```sql
CREATE TABLE premium_cost_ledger (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES users(id),
  app_slug text NOT NULL DEFAULT 'launch-studio-pro',
  project_id uuid REFERENCES launch_projects(id),
  run_id uuid REFERENCES premium_app_runs(id),
  estimated_cost_usd numeric,
  actual_cost_usd numeric,
  created_at timestamptz DEFAULT now()
);
```

### Alertas (PRD §10)
```ts
// Verificação pós-run
const dailyTotal = await getDailyCost(new Date());
if (dailyTotal > 50) {
  logger.warn('COST_ALERT: Daily cost exceeded $50', { date, total: dailyTotal });
}
```

### Dependência
Depende de 4.4 (cost tracker e `estimateCost`), 4.1 (tabela `premium_app_runs`).

### Testing

- Run concluída: ledger tem `actual_cost_usd` populado
- Run falhada: ledger tem `actual_cost_usd = null` ou 0
- `getDailyCost` > 50: WARN log gerado
