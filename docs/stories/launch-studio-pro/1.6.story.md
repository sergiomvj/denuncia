## Story

**As a** desenvolvedor da Suite Sextou.biz,
**I want** inserir o `launch-studio-pro` na tabela `mini_apps` e no manifest do app,
**so that** o Launch Studio PRO seja reconhecido pela suite como um app premium ativo com seus artifact_types listados (PRD §13).

## Acceptance Criteria

1. Registro inserido em `mini_apps` com campos: `slug='launch-studio-pro'`, `name='Launch Studio PRO'`, `suite='Sextou.biz Premium Apps'`, `requires_premium=true`, `is_active=true` (PRD §8.1 tabela `mini_apps`).
2. Arquivo de manifest `app-manifest.yaml` (ou equivalente no projeto) contém a configuração completa do app conforme PRD §13.
3. `artifact_types` listados no manifest: `launch_dossier_pdf`, `email_pack`, `video_scripts`, `sales_page_html`, `calendar_ics`, `checklist_markdown`, `triggers_map` (PRD §13).
4. Campos de LLM no manifest: `mode=premium_multi_model`, `routing_profile=quality_first`, `default_language=pt-BR` (PRD §13).
5. Campos de agentes no manifest listam os 10 agentes runtime: `interview_agent`, `avatar_agent`, `offer_architect`, `sequence_planner`, `trigger_mapper`, `scarcity_designer`, `copywriter_agent`, `jl_partner_advisor`, `fidelity_reviewer`, `export_agent` (PRD §6, §13).
6. Insert em `mini_apps` é idempotente (`INSERT ... ON CONFLICT (slug) DO NOTHING` ou equivalente).
7. Query `SELECT * FROM mini_apps WHERE slug = 'launch-studio-pro'` retorna 1 linha com `requires_premium=true` e `is_active=true`.
8. `npm run lint` e `npm run typecheck` passam.

### Conteúdo completo do manifest (PRD §13)

```yaml
app:
  name: "Launch Studio PRO"
  slug: "launch-studio-pro"
  suite: "Sextou.biz Premium Apps"
  category: "Marketing & Estratégia de Lançamento"
  requires_premium: true
  access_gate: "users.is_premium = TRUE"
  artifact_types:
    - launch_dossier_pdf
    - email_pack
    - video_scripts
    - sales_page_html
    - calendar_ics
    - checklist_markdown
    - triggers_map

  mobile_first: true

llm:
  mode: premium_multi_model
  routing_profile: quality_first
  default_language: pt-BR
  supported_languages: [pt-BR, en-US, es-ES]

    - sequence_planner
    - trigger_mapper
```
