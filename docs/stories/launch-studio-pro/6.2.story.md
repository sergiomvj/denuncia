## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** criar as tabelas `generated_artifacts` e `artifact_versions` e implementar upload de artefatos no storage (Cloudflare R2 ou Supabase Storage),
**so that** PDFs, ZIPs e MDs gerados sejam persistidos com caminho previsível e metadados acessíveis (params §11.7-8, §13.2; PRD §7).

## Acceptance Criteria

1. Tabela `generated_artifacts` criada com campos: `id`, `dossier_id` (FK), `user_id`, `artifact_type` (pdf/zip/md/docx/ics/csv), `storage_path`, `file_size_bytes`, `created_at`.
2. Tabela `artifact_versions` criada com campos: `id`, `artifact_id` (FK), `version`, `storage_path`, `created_at`.
3. Caminho de storage previsível: `{user_id}/launch-studio-pro/{project_id}/v{version}/{artifact_type}.{ext}` (params §13.2).
4. Suporte a Cloudflare R2 como storage primário com fallback para Supabase Storage (PRD §7).
5. Módulo `lib/launch-studio-pro/storage.ts` com função `uploadArtifact(content: Buffer, meta: ArtifactMeta): Promise<string>` que retorna a URL pública/signed URL.
6. `generated_artifacts.storage_path` salvo após upload bem-sucedido.
7. Testes de upload mockados (sem chamada real ao R2).
8. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Caminho de storage (params §13.2)
```
{user_id}/launch-studio-pro/{project_id}/v{version}/{artifact_type}.{ext}
```
Exemplo: `user123/launch-studio-pro/proj456/v1/dossier.pdf`

### R2 via SDK S3-compatible

```ts
import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
const r2 = new S3Client({
  region: 'auto',
  endpoint: `https://${process.env.CLOUDFLARE_R2_ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: process.env.CLOUDFLARE_R2_ACCESS_KEY_ID!,
    secretAccessKey: process.env.CLOUDFLARE_R2_SECRET_ACCESS_KEY!,
  },
});
```
