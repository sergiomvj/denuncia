## Story

**As a** usuário premium do Launch Studio PRO,
**I want** um wizard de entrevista guiada com 10 etapas navegáveis, barra de progresso e capacidade de salvar e retomar,
**so that** eu possa responder as perguntas PLF no meu ritmo sem perder progresso (PRD §4.2, §3.5).

## Acceptance Criteria

1. Componente wizard renderiza as 10 etapas em sequência, uma por vez, com barra de progresso mostrando etapa atual/total (ex: "Etapa 3 de 10").
2. Navegação: botões "Próxima" (avança) e "Anterior" (volta) funcionam sem perder dados da etapa atual.
3. A etapa atual é salva automaticamente (auto-save) ao clicar "Próxima" ou ao perder foco do campo principal (PRD §8.2 persistência).
4. Usuário pode fechar o wizard e retomar de onde parou — o wizard carrega o `step_number` mais recente salvo em `launch_interview_answers`.
5. Etapas são configuráveis via array de configuração (não hardcoded no componente wizard — a definição das 10 etapas vem da story 3.2).
6. Barra de progresso é visual e indica claramente etapas completadas vs. pendentes.
7. Layout mobile-first, Design System v2.
8. Estado de loading durante auto-save é visível (ex: spinner/ícone de sync).
9. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### Arquitetura do wizard

```tsx
// Estrutura sugerida
interface WizardStep {
  stepNumber: number;
  stepSlug: string;
  title: string;
  component: React.ComponentType<StepProps>; // definido em 3.2
}

interface InterviewWizardProps {
  steps: WizardStep[];
  projectId: string;
  initialAnswers: Record<number, any>; // respostas salvas
  onComplete: (allAnswers: Record<number, any>) => void;
}
```

### API de persistência

```ts
// UPSERT por project_id + step_number
await prisma.launchInterviewAnswers.upsert({
  where: { projectId_stepNumber: { projectId, stepNumber } },
  update: { answers, coachTipShown },
  create: { projectId, stepNumber, stepSlug, answers, coachTipShown }
});
```

Nota: adicionar constraint unique `(project_id, step_number)` em `launch_interview_answers` se não existir.

### Dependência
Depende de 2.3 (projeto criado com `project_id`), 1.1 (tabela `launch_interview_answers`).

### A story 3.2 é necessária para o conteúdo das etapas
Esta story cria o MOTOR (wizard engine). Os campos de cada etapa são definidos em 3.2.

### Testing
