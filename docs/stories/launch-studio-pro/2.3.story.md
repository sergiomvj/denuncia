## Story

**As a** usuário premium do Launch Studio PRO,
**I want** preencher um briefing inicial com apenas 3 campos (Nome do Projeto, Público-Alvo e Ideia do Produto),
**so that** eu possa iniciar um novo projeto de lançamento de forma rápida e avançar para a entrevista guiada (PRD §4.1, Fluxo passo 4).

## Acceptance Criteria

1. Tela/modal de briefing inicial exibe exatamente 3 campos: "Nome do Projeto" (texto curto), "Público-Alvo" (texto curto), "Ideia do Produto & Campanha" (textarea) — conforme PRD §4.1.
2. "Ideia do Produto & Campanha" exige mínimo de 80 caracteres (PRD §4.1).
3. Todos os 3 campos são obrigatórios.
4. Validação client-side mostra erros inline (não toast genérico) antes do submit.
5. Ao submeter formulário válido, chama `POST /api/launch-studio-pro/projects` (criado em 2.2) e persiste dados em `launch_projects` com `status='draft'`.
6. Após criação bem-sucedida, redireciona para a entrevista guiada (rota da story 3.1) passando o `project_id`.
7. Campos têm placeholders de exemplo conforme PRD §4.1: Nome ex. "Lançamento Curso Violão Zero", Público ex. "Homens 35-55 que querem tocar violão".
8. Layout mobile-first, Design System v2.
9. `npm run lint` e `npm run typecheck` passam.
