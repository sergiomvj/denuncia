## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** registrar tokens consumidos, custo estimado/real e latência por chamada LLM, acumulando o total em `premium_app_runs`,
**so that** o sistema possa exibir e controlar custos por projeto/usuário (params §6, §11.5; PRD §10).

## Acceptance Criteria

1. Após cada chamada LLM bem-sucedida, `premium_llm_calls` é atualizado com: `input_tokens`, `output_tokens`, `cost_usd` (calculado via tabela de preços por provedor/modelo), `latency_ms`.
2. `premium_app_runs.total_cost_usd` é incrementado com o custo de cada chamada da run.
3. `premium_app_runs.total_tokens` é incrementado com os tokens de cada chamada.
4. Tabela de preços por modelo (`premium_model_configs.cost_per_input_token`, `cost_per_output_token`) adicionada à migration de 4.1 ou nova migration.
5. Estimativa de custo calculada antes de iniciar a geração: função `estimateCost(phaseRole[])` retorna valor em USD baseado nos tokens médios esperados.
6. Estimativa exibida no modal de confirmação de 3.5 (substituindo o placeholder ~$0.68).
7. `npm run lint`, `npm run typecheck` e testes de cálculo de custo passam.
