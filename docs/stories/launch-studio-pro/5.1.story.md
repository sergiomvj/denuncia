## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** criar o system prompt base da PLF e os prompt contracts separados por agente,
**so that** todos os agentes operacionais sigam as regras inegociáveis da Product Launch Formula™ de Jeff Walker (PRD §9.1, Prompt.txt; params §10).

## Acceptance Criteria

1. Arquivo `lib/launch-studio-pro/prompts/system-prompt-base.ts` criado com o system prompt base completo do PRD §9.1 (8 regras inegociáveis).
2. Prompt contracts separados por agente criados em `lib/launch-studio-pro/prompts/` — um arquivo por agente: `avatar-agent.ts`, `offer-architect.ts`, `sequence-planner.ts`, `trigger-mapper.ts`, `scarcity-designer.ts`, `copywriter-agent.ts`, `jl-partner-advisor.ts`, `fidelity-reviewer.ts`, `export-agent.ts`.
3. Cada prompt contract inclui: descrição do agente, inputs esperados, outputs esperados, regras específicas do agente.
4. System prompt base é prefixado automaticamente a todos os contratos de agente (não duplicado em cada arquivo).
5. Regra de saída em JSON validável presente no system prompt: "Saída sempre em JSON validável pelo schema fornecido." (PRD §9.1 regra 8).
6. Regra de aviso de compliance presente: "Nunca prometa resultados financeiros específicos." (PRD §9.1 regras 5-6).
7. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### System prompt base (PRD §9.1 — transcrição fiel)

```
Você é um estrategista sênior da Launch Studio PRO (Suite Sextou.biz Premium),
especializado na Product Launch Formula™ de Jeff Walker. Sua missão é ajudar
empreendedores brasileiros a estruturar lançamentos que geram expectativa,
diálogo e conversão — nunca "Marketing da Esperança".

Regras inegociáveis:
1. Fidelidade total à PLF: Pré-Pré → PLC 1 (Oportunidade) → PLC 2 (Transformação) → PLC 3 (Propriedade) → Abertura → Fechamento.
2. Toda oferta deve responder: "Qual a transformação final?"
3. Todo lançamento precisa de escassez REAL (preço, bônus ou encerramento).
4. Estímulos mentais devem ser distribuídos em camadas, nunca isolados.
5. Nunca prometa resultados financeiros específicos.
6. Sempre inclua aviso: "Resultados variam conforme execução e mercado."
7. Responda em pt-BR, tom humano, direto, sem linguagem corporativa.
8. Saída sempre em JSON validável pelo schema fornecido.
```

### Dependência
Depende de 4.2 (roteador usa os prompts para construir as chamadas).

### Article IV — No Invention
Os prompt contracts descrevem apenas o que o PRD §6 especifica por agente. Não adicionar capacidades que não estejam no PRD.

### Testing

- `getSystemPromptBase()` contém as 8 regras
- `buildPrompt('avatar_agent', context)` inclui system prompt base + contrato do agente
- Nenhum arquivo de prompt contém API key ou dado sensível
