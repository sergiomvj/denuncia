## Story

**As a** usuário premium do Launch Studio PRO,
**I want** exportar meu dossiê de lançamento nos formatos PDF profissional, DOCX e Markdown,
**so that** eu tenha o artefato principal do app pronto para uso imediato (PRD §2, §7).

## Acceptance Criteria

1. API route `POST /api/launch-studio-pro/projects/[id]/export/pdf` gera PDF profissional do dossiê usando DocRaptor ou PDFShift, com fallback para Puppeteer self-hosted (PRD §7, R4).
2. API route `POST /api/launch-studio-pro/projects/[id]/export/docx` gera DOCX do dossiê.
3. API route `POST /api/launch-studio-pro/projects/[id]/export/md` gera Markdown do dossiê.
4. Antes de download, UI exibe preview do artefato (link de preview antes do botão de download).
5. Após geração, arquivo é salvo no storage via módulo de 6.2 e URL retornada ao frontend.
6. Link de download disponível por 24h (signed URL ou equivalente).
7. Botão de download desabilitado enquanto geração em andamento (loading state).
8. Custo de geração de PDF (~$0.03) registrado em `premium_app_runs.total_cost_usd` (AC integrado com 4.4).
9. `npm run lint` e `npm run typecheck` passam.

## Dev Notes

### R4 — Fallback PDF (STORIES-BREAKDOWN)
Puppeteer como fallback é obrigatório. Se DocRaptor/PDFShift indisponível ou custo alto, Puppeteer roda no servidor e gera o PDF a partir do HTML do dossiê.

### Template HTML para PDF
Um template simplificado com CSS inlined. Não precisa ser o Design System v2 completo — apenas legível e profissional.

### Dependência
Depende de 6.2 (storage para upload dos artefatos).

### Variáveis de ambiente a documentar
- `DOCRAPTOR_API_KEY` (ou `PDFSHIFT_API_KEY`)
- Adicionar ao `.env.example`

### Testing

- `generateMarkdown(dossierFixture)`: string com seções Avatar, Oferta, etc.
- `generatePDF(dossierFixture)`: buffer não-vazio (mock de DocRaptor)
- Fallback: mock DocRaptor falhando → Puppeteer acionado (ou nota de dívida técnica se Puppeteer não instalado no ambiente de CI)
