## Story

**As a** desenvolvedor do Launch Studio PRO,
**I want** criar as tabelas de orquestração premium (`premium_model_providers`, `premium_model_configs`, `premium_app_runs`, `premium_llm_calls`, `premium_tool_calls`) via migration,
**so that** o orquestrador multi-LLM tenha persistência para configuração de provedores, rastreamento de execuções e custo por chamada (params §11).

## Acceptance Criteria

1. Tabela `premium_model_providers` criada com campos: `id`, `provider_name` (anthropic/openai/google/perplexity), `is_active`, `created_at`.
2. Tabela `premium_model_configs` criada com campos: `id`, `provider_id` (FK), `model_name`, `phase_role` (interview_coach/planner/copywriter/avatar_researcher/objection_handler/reviewer/exporter), `routing_profile` (quality_first), `max_tokens`, `temperature`, `is_active`, `created_at`.
3. Tabela `premium_app_runs` criada com campos: `id`, `user_id` (FK), `app_slug`, `project_id` (FK para `launch_projects`), `status` (pending/running/completed/failed), `started_at`, `completed_at`, `total_cost_usd`, `total_tokens`.
4. Tabela `premium_llm_calls` criada com campos: `id`, `run_id` (FK), `model_config_id` (FK), `phase_slug`, `attempt_number`, `input_tokens`, `output_tokens`, `cost_usd`, `latency_ms`, `status` (success/failed/fallback), `created_at`.
5. Tabela `premium_tool_calls` criada com campos: `id`, `run_id` (FK), `tool_name`, `input_summary`, `output_summary`, `created_at`.
6. Seed de provedores e configs por fase/role conforme PRD §5 inserido.
7. `npx prisma validate` e `npm run typecheck` passam.
